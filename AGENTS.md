# AGENTS.md

## 작업 원칙

1. 항상 "동작하는 코드"를 만든다
2. 파일 단위로 완성된 결과를 제공한다
3. import 오류가 없도록 한다
4. 실행 가능한 상태를 유지한다
5. 하드코딩 최소화 (env 사용)
6. 구조를 먼저 설계하고 구현한다

## 개발 방식

작업 순서:
1. 디렉토리 구조 정의
2. DB 모델 정의
3. 핵심 API 구현
4. 서비스 로직 분리
5. 프론트 연결
6. 테스트 코드 추가

## 코드 스타일

- 함수는 단일 책임 원칙 유지
- 서비스 레이어 분리
- 비즈니스 로직은 route에 넣지 않는다
- 설정값은 config 또는 env로 관리

## FDS 관련 규칙

- 모든 거래는 risk evaluation을 거친다
- risk engine은 독립 모듈로 유지한다
- rule은 확장 가능하게 작성한다
- 탐지 결과는 반드시 DB에 저장한다

## 금지사항

- 단순 예제 코드
- 하드코딩된 테스트 값 남기기
- 실행 불가능한 코드
- 임시방편 코드

## 출력 요구사항

- 전체 파일 형태로 제공
- 필요한 경우 여러 파일을 나눠서 제공
- 실행 방법 포함
