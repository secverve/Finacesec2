# 기능 설명서

## 개요

VERVE FDS Infra는 `국내주식 HTS UI`와 `FDS 관제/실습 환경`을 함께 제공하는 모의 증권 시스템입니다.

- 트레이더는 실시간 시세, 차트, 주문, 체결, 잔고, 수익률을 확인할 수 있습니다.
- 관리자는 관제형 HTS 화면에서 위험 이벤트를 모니터링하고 직접 조치할 수 있습니다.
- 모든 주문은 FDS 평가를 거친 뒤 처리되며, 결과는 이벤트/룰 히트/감사 로그로 저장됩니다.

## 사용자 역할

### 1. 트레이더

- 한국형 HTS 스타일 화면 사용
- 종목 검색 및 현재가 조회
- 호가창, 체결창, 분봉/일봉 차트 조회
- 시장가/지정가 주문
- 주문 상태 확인: `접수`, `보류`, `차단`, `체결`
- 보유 종목, 평가손익, 수익률 확인

### 2. 관리자

- 관제형 HTS 화면 사용
- 실시간 위험 이벤트 큐 확인
- 사건 상세 분석
- 감사 로그 추적
- FDS 룰 카탈로그 조회
- 공격 실습 시나리오 실행
- 위험 이벤트에 대해 `승인`, `차단`, `추가인증`, `계정잠금`, `잠금해제` 조치 수행

### 3. 애널리스트

- 현재 구조상 로그인 가능 계정으로 제공
- 관리자와 유사한 보안/운영 화면 확장에 활용 가능한 역할

## 트레이더 HTS 기능

### 시세/시장 화면

- 상단 탭 기반 HTS 화면
- 제공 탭
  - `현재가(1)`
  - `복수 현재가(1)`
  - `시간외 체결(1)`
  - `일자별 주가(1)`
  - `호가잔량 추이`
  - `당일/전일 주가비교`
- 시장 필터 지원
  - `통합`
  - `KRX`
  - `NXT`
  - `관심`

### 차트

- 공개 시세 API 기반 실시간 차트 데이터 조회
- 지원 주기
  - `1분`
  - `3분`
  - `5분`
  - `10분`
  - `일봉`
- 캔들 차트 표시
- 거래량 표시
- 이동평균선 표시
- 캔들 선택 시 시가/고가/저가/종가/거래량 확인

### 주문/체결

- 시장가 주문
- 지정가 주문
- 지정가 주문은 시세 충족 시 자동 체결
- 체결 시간, 체결 수량, 미체결 수량 추적
- 주문 상태 변화가 잔고/수익률에 반영

### 계좌/수익률

- 예수금
- 총자산
- 보유 종목
- 평가손익
- 수익률
- 관심종목 개수 및 주문 현황

### 단축키

- `F5`: 전체 새로고침
- `Alt+1`: 주문 탭
- `Alt+2`: 분봉/일봉 탭
- `Alt+3`: FDS 경보 탭
- `Alt+4`: 잔고 탭
- `Alt+B`: 매수 전환
- `Alt+S`: 매도 전환

## 관리자 관제형 HTS 기능

### 실시간 관제

- 위험 이벤트 큐
- 선택 이벤트 상세 요약
- 점수, 심각도, 판정, 지역, 디바이스 확인
- 이벤트 선택 시 중앙 분석 화면과 우측 대응 콘솔 연동
- 세션/단말/추가인증 현황 요약

### 사건 분석

- 이벤트 기본 정보
- 룰 히트 상세
- 관리자 조치 이력
- 관련 감사 로그 추적

### 대응 콘솔

- 즉시 대응 버튼 제공
  - `승인`
  - `차단`
  - `추가인증`
  - `계정잠금`
  - `잠금해제`
- 선택 사건 기준 연관 엔터티 확인
  - 이벤트 ID
  - 주문 ID
  - 사용자 ID
  - IP
  - 지역
  - 디바이스

### 운영/감사 화면

- 감사 로그 목록
- 요청 추적 ID 확인
- 조치 결과 확인
- 최근 실습 결과 확인
- 룰 카탈로그 확인
- FDS 전용 탐지 로그 창
- 상관분석 로그 스트림

### 세션 보안

- 로그인 시 세션 발급 및 세션 ID 바인딩
- 활성 세션 목록 조회
- 세션 위험 점수 조회
- 인증 강도 조회
- 관리자 세션 회수
- 단말 바꿔치기 토큰 사용 차단

### 단말 신뢰

- 단말 식별 및 fingerprint 기반 관리
- 신뢰/관찰/추가인증/차단 상태 관리
- 관리자 단말 신뢰 승인
- 관리자 단말 차단
- 차단 단말의 활성 세션 회수

### 보안 정책 카탈로그

- 로그인 실패 잠금 정책
- 신규 단말 단계인증 정책
- 동시 세션 모니터링 정책
- 고액 주문 step-up 정책
- 추적 ID 기반 감사 정책

## FDS 기능

### 기본 원칙

- 모든 거래는 주문 생성 직후 FDS 평가 수행
- 평가 결과는 반드시 DB에 저장
- 탐지 결과는 확장 가능한 룰 기반으로 계산
- FDS 결과 이후에도 세션/단말 보안 통제를 추가 적용

### 주요 탐지 포인트

- 신규 디바이스 고액 주문
- 해외/고위험 지역 접속 후 주문
- 로그인 실패 누적 후 주문
- 고액 주문 급증
- 짧은 시간 내 주문 폭주
- 감시 종목 주문
- 동일 IP 다계정 주문

### 평가 결과

- `ALLOW`
- `HELD`
- `BLOCKED`
- `AUTH_REQUIRED`

### 저장 데이터

- `Risk Event`
- `Rule Hit`
- `Admin Action`
- `Audit Log`

## 공격 실습 기능

관리자 화면에서 안전한 모의 시나리오를 실행할 수 있습니다.

- 감시종목 고액 주문
- 계정대입 후 주문 시도
- 동일 IP 다계정 거래 패턴

실습 목적:

- FDS가 어떤 조건에서 탐지하는지 확인
- 관리자 조치 흐름 연습
- 감사 로그와 연계된 포렌식 관찰

## 로그/감사 구조

- 구조화된 감사 로그 저장
- 요청 단위 추적 ID 기록
- 주문 생성, 로그인, 관리자 조치, 실습 실행 이력 저장
- 관리자 관제 화면에서 관련 로그 추적 가능

## 실시간 데이터와 시뮬레이션 범위

- 현재가/차트는 공개 시세 API를 사용합니다.
- HTS 화면 일부 보조 지표와 호가/체결 보드는 학습용 시뮬레이션 로직을 포함합니다.
- 즉, `실제 공개 시세 + 모의 주문/FDS 엔진`이 결합된 형태입니다.
